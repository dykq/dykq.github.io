关于grunt/gulp/webpack:
什么时候使用grunt/glup：
#如果一个工程的模块化依赖很简单或者没有用到模块化的概念
#只需要进行简单的合并、压缩，就可以使用grunt/gulp即可。
#若是项目需要使用模块化管理，而且相互依赖非常强时需要用到webpack。
gulp的task：
#下面的task就是将src下面所有的js文件转成es5语法并且最终输出到dist文件夹中
gulp.task(‘js’,() =>
gulp.src(‘src/*.js)
.pipe(babel({
presets:[‘es2015’]
}))
.pipe(gulp.dest(‘dist’))
);
区别：
gulp和grunt强调的是前段流程的自动化而webpack强调的是模块化开发管理，文件压缩合并、预处理等功能只是webpack附带功能。
}
src开发的目录 ->打包后放入dist(发布的简写) distribution
如何用webpack进行打包 只需要在相对应的目录下 webpack 需要打包的目录(只需要一个，当做入口) 打包后文件的目录以及名字

模块化写法 使用闭包去实现模块化
一般写法： 将所需要的文件封装成一个闭包用一个全局变量去接收这个闭包 在闭包中定义个空对象 并且将所需要的东西导出赋值给一个全局变量中(比如命名为model)，需要用到的方法变量在函数中用一个空的对象去存放并导出 在另一个地方使用时只需要model.即可 如：
;var model = (function(){
var res = {};
var name = “model”;
var flag = true;
function sum(num1,num2){
return num1+num2;
}
res.sum = sum;
res.name = name;
res.flag = flag;
return res;
}())
这样就将res导出了，并在使用时以model.来使用 比如model.flag 去获取这个flag是true或者false

commonjs写法：
在导出的时候：module.exports = {
var flag = true;
function sum(num1,num2){
return num1+num2;
}
}
导入：let{flag,sum} = require(‘地址’)

es6中的模块化：
在script中添加type = “module”可以使该js模块化不会被其他js所影响
在需要导出的js文件中写export{里面写需要传出去的名称以及函数} 或者在定义一个变量或者函数时定义可以导出
如：exoprt let name = “zch”;export function(){}就可以
在需要到如的js文件中写import{里面写需要用到的数据以及方法} from “所需数据的js地址.js”;
特殊情况 当你定义了一个const变量或者函数想要导出时还可以使用default，一个模态化的js中只能存在一个export default，切记！
并且在需要改变量时候只需要 import abc(ps：随便命名) from ‘地址.js’;如：
const address = “北京”; export default address; 或者用default导出一个函数
export default function(){}
在导入时：import abc from “地址.js” 其中abc是任意取名
导入的时候若全部导入或者导入对象较多时候可以使用 import * as aaa from “地址.js”的方式将数据存放在aaa对象中 在以aaa.对应数据的方式引用 其中aaa为任意命名
该语法只在script 的type是module时候可用！！
