父子组件的通信：

使用props来声明需要从父组件中接收的数据 在创建的组件出接收 用:props中的名字="data中的数据"
      那么props中的相对应变量就会存放实例data中的数据了
    props有两种写法 一种是字符串数组的形式来传递名称 如： props: ['cfruit', 'cmessage']
    另一种是传递一个对象，对象时还需要定义他的类型 如：
     props:{cfruit:Array,cmessage:String}
    常见类型：String、Number、Boolean、Object、Data、Function、Symbol，也可以是自定义类型
  在传递一个对象时还可以对其设置默认值 如：
   props:{cfruit:{
              type:Array,
              default(){
               return ["没有水果了"]
              }
                },
          cmessage:{
               type:String,
               default:"sdfsdfdsf"
                    }}
                    当类型是Object或者Array是需要返回一个function,若数据是多种类型可以用数组表示如：
                      cfruit:{
                          type:[Array,Number],
                      }
                      还有一个required 若required为true 则必须给他传值,
                      validator自定义函数确定接收到的数据必须是下面几种 如：
                       cfruit:{
                         validator：function(value){
                           return ['success','warning','danger'].indexof(value) !== -1
                         }
                       }
                       此时cfruit从父组件中传过来的值必须是success、warning或者danger

子组件中的按钮发生了点击后 父组件监听需要用到 this.$emit(“类型”，数据)
在引用这个模板中写个自定义的点击事件@类型与$emit 中的类型一致:方法名–不需要括号
在 Vue 实例 methods 中方法名(数据) 他会默认接收子组件传过来的数据

为什么组件中的 data 返回的是一个函数 因为如果不是函数的话 其他位置调用这个组件 比如这个组件是一个计数器 其他计数器加减时候 若不是函数而是如同实例中的 data 写法，就会指向同一个对象，就会引起连锁反应让所有的计数器组件中的计数相同， 这样是不好的 而当他是一个函数的时候 每个计数器指向的内存地址是不同的 就不会有连锁反应

其实组件的原型是指向 Vue 的, Vue 里面拥有的属性基本组件中存在

数组使用的三种方法 ：
            filter(function(n){}) 需要返回的是true or false 可以过滤数值 比如提取出数组中小于一百的数值，返回到一个新的数组，其中n为数组中对应的值
            map(function(n){})  返回一个数值 对数组中的数值进行加减乘除  如对数组中的每一个数值乘以二  n为数组中对应的值
            reduce(function(previous,n){return previous+n},初始化值)    对数组中的数值进行汇总 其中previous一开始的值为初始化的值，之后previous为previous+n ,n为数组相对应的数值 就可以计算出数组的所有值的总和

模块化写法 使用闭包去实现模块化
一般写法： 将所需要的文件封装成一个闭包用一个全局变量去接收这个闭包 在闭包中定义个空对象 并且将所需要的东西导出赋值给一个全局变量中(比如命名为model)，需要用到的方法变量在函数中用一个空的对象去存放并导出 在另一个地方使用时只需要model.即可 如：
;var model = (function(){
var res = {};
var name = “model”;
var flag = true;
function sum(num1,num2){
return num1+num2;
}
res.sum = sum;
res.name = name;
res.flag = flag;
return res;
}())
这样就将res导出了，并在使用时以model.来使用 比如model.flag 去获取这个flag是true或者false

commonjs写法：
在导出的时候：module.exports = {
var flag = true;
function sum(num1,num2){
return num1+num2;
}
}
导入：let{flag,sum} = require(‘地址’)

es6中的模块化：
在script中添加type = “module”可以使该js模块化不会被其他js所影响
在需要导出的js文件中写export{里面写需要传出去的名称以及函数} 或者在定义一个变量或者函数时定义可以导出
如：exoprt let name = “zch”;export function(){}就可以
在需要到如的js文件中写import{里面写需要用到的数据以及方法} from “所需数据的js地址.js”;
特殊情况 当你定义了一个const变量或者函数想要导出时还可以使用default，一个模态化的js中只能存在一个export default，切记！
并且在需要改变量时候只需要 import abc(ps：随便命名) from ‘地址.js’;如：
const address = “北京”; export default address; 或者用default导出一个函数
export default function(){}
在导入时：import abc from “地址.js” 其中abc是任意取名
导入的时候若全部导入或者导入对象较多时候可以使用 import * as aaa from “地址.js”的方式将数据存放在aaa对象中 在以aaa.对应数据的方式引用 其中aaa为任意命名
该语法只在script 的type是module时候可用！！

关于grunt/gulp/webpack:
什么时候使用grunt/glup：
#如果一个工程的模块化依赖很简单或者没有用到模块化的概念
#只需要进行简单的合并、压缩，就可以使用grunt/gulp即可。
#若是项目需要使用模块化管理，而且相互依赖非常强时需要用到webpack。
gulp的task：
#下面的task就是将src下面所有的js文件转成es5语法并且最终输出到dist文件夹中
gulp.task(‘js’,() =>
gulp.src(‘src/*.js)
.pipe(babel({
presets:[‘es2015’]
}))
.pipe(gulp.dest(‘dist’))
);
区别：
gulp和grunt强调的是前段流程的自动化而webpack强调的是模块化开发管理，文件压缩合并、预处理等功能只是webpack附带功能。
